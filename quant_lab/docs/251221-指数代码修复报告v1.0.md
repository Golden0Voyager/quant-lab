```python
def detect_asset_type(code):
    # 指数: 1A(上证), 1B(上证), 399(深证), 000(上证),
    #      931/930(中证), H30(港股), sh/sz开头
    if code.startswith(("1A", "1B", "000", "399", "931", "930",
                       "H30", "sh000", "sz399")) or code == "000001":
        return "index"
    # ETF/LOF: 15/16(深), 51/56/58(沪)
    if code.startswith(("15", "16", "51", "56", "58")):
        return "etf"
    return "stock"

def clean_code_for_akshare(code, asset_type):
    if asset_type == 'index':
        # 上证指数处理 (000开头的上证指数)
        if code.startswith("000") and len(code) == 6:
            return f"sh{code}"
        # 上证指数处理 (特殊格式)
        if code == "1A0001" or code == "000001":
            return "sh000001"
        # 深证/创业板/中证指数处理 (399开头)
        if code.startswith("399"):
            return f"sz{code}"
    return code
```

## 🎯 技术要点总结

### AkShare 指数代码规则

1. **深证指数** (399xxx)
   - 输入格式: `399050`, `399441`, `399971`, `399976`
   - AkShare格式: `sz399050`, `sz399441`, `sz399971`, `sz399976`
   - 转换规则: 前缀添加 `sz`

2. **上证指数** (000xxx)
   - 输入格式: `000001`, `000932`
   - AkShare格式: `sh000001`, `sh000932`
   - 转换规则: 前缀添加 `sh`

3. **中证指数** (931xxx/930xxx)
   - 说明: 这些是中证指数公司的内部代码
   - 处理: 需要映射到对应的Wind代码（399xxx或000xxx）
   - 示例: 931151 → 399971, 930721 → 399976

4. **港股指数** (H30xxx)
   - 说明: AkShare暂不支持港股指数
   - 处理: 需要映射到对应的A股相关指数
   - 示例: H30533 → 399050 (中证互联网)

### 代码设计原则

1. **资产类型识别** (detect_asset_type)
   - 尽可能扩展识别模式，避免误判为股票
   - 支持多种指数代码格式（1A/1B/000/399/931/930/H30）

2. **代码清洗** (clean_code_for_akshare)
   - 根据资产类型进行不同的格式转换
   - 上证指数加 `sh` 前缀
   - 深证指数加 `sz` 前缀
   - 股票和ETF保持原样

3. **容错机制**
   - 如果检测到无效代码，会记录错误日志
   - 不会影响其他标的的数据抓取
   - 在报告中明确标记缺失状态
