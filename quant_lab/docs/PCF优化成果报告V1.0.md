## 🔧 技术实现

### 1. 新增方法: _fetch_operating_cashflow_direct()

```python
def _fetch_operating_cashflow_direct(self, symbol: str) -> Optional[float]:
    """
    直接获取经营活动现金流净额

    数据源: ak.stock_financial_report_sina()
    优势: 直接从财报获取，无需反推
    """
    # 转换股票代码格式（sh600519 或 sz000001）
    sina_symbol = f'sh{symbol}' if symbol.startswith(('600', '601', '603', '688')) else f'sz{symbol}'

    # 获取现金流量表
    df = ak.stock_financial_report_sina(stock=sina_symbol, symbol='现金流量表')

    # 提取最新有效数据
    target_col = '经营活动产生的现金流量净额'
    valid_data = df[df[target_col].notna()]

    if not valid_data.empty:
        latest_cf = float(valid_data.iloc[0][target_col])
        return latest_cf

    return None
```

**关键优势**:
- ✅ 数据来源权威（新浪财经 → 交易所公告）
- ✅ 无需复杂反推逻辑
- ✅ 自动处理负现金流
- ✅ 支持沪深两市

## ⚠️ 已知限制

### 1. P/FCF暂时N/A

**原因**:
```
capex_col = '购建固定资产、无形资产和其他长期资产支付的现金'
```
该列名在新浪财报中可能为：
- ✅ "购建固定资产、无形资产和其他长期资产支付的现金" (标准)
- ❓ "购建固定资产、无形资产和其他长期资产所支付的现金" (变体)
- ❓ "购置固定资产、无形资产和其他长期资产支付的现金" (变体)

**解决方案**:
```python
# 模糊匹配列名
capex_cols = [col for col in df.columns if '购' in col and '固定资产' in col and '支付' in col]
if capex_cols:
    capex_col = capex_cols[0]  # 使用第一个匹配的列
```

**原因**: 财务报表不完整或延迟披露

**解决方案**: 已有降级逻辑，返回N/A
