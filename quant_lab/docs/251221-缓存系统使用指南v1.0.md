## 🔧 缓存策略配置

### 默认TTL配置

| 数据类型 | 更新频率 | 缓存有效期 | 说明 |
|----------|----------|------------|------|
| 估值数据 (PE/PB/PS) | 每日 | 24小时 | 盘后更新 |
| 业绩数据 (财报) | 每季度 | 7天 | 财报发布后更新 |
| 实时行情 (量比/换手率) | 实时 | 5分钟（盘中）<br>24小时（盘后） | 交易时间内短缓存 |
| 资金流向 | 每日 | 24小时 | 日度统计 |
| 股东数据 | 每季度 | 30天 | 季报披露 |
| 北向资金 | 每日 | 24小时 | 每日统计 |
| 新闻舆情 | 实时 | 1小时 | 时效性强 |
| 汇率数据 | 实时 | 1小时 | 外汇波动 |
| ETF折溢价 | 实时 | 5分钟 | 实时套利机会 |

### 修改TTL

```python
from data_cache import CacheStrategy

# 临时修改（当次有效）
CacheStrategy.TTL_DAY = 12 * 60 * 60  # 改为12小时

# 永久修改：编辑 data_cache.py
class CacheStrategy:
    TTL_DAY = 12 * 60 * 60  # 改为12小时
```

    print(f"{point['date']}: PE={point['pe_ttm']}, PB={point['pb']}")

## 🐛 故障排查

### 问题1：缓存不生效

**现象：** 每次都重新抓取数据

**排查：**
```python
from analyst_cached import get_cache_info

# 查看缓存是否正常写入
stats = get_cache_info()

if stats['total_cache'] == 0:
    print("缓存未写入，检查权限")
```

### 问题2：数据过时

**解决方案：**
```python
# 方式1：删除缓存数据库
rm quant_cache.db

# 方式2：手动清理
from analyst_cached import clear_cache
clear_cache()
```

### 问题3：磁盘空间

**缓存大小估算：**
- 单只股票：约 5-10 KB
- 100只股票：约 500 KB - 1 MB
- 1000只股票：约 5-10 MB

**清理建议：**
- 定期清理过期缓存（自动）
- 删除历史数据点（手动）

```python
# 清理超过1年的历史数据
import sqlite3
conn = sqlite3.connect("quant_cache.db")
cursor = conn.cursor()
cursor.execute("DELETE FROM historical_valuation WHERE date < date('now', '-365 days')")
conn.commit()
conn.close()
```
