## 系统概述

### 🏗️ 系统架构

Quant Lab V2.0 采用**双层 AI 架构**：

```
┌─────────────────────────────────────────────────┐
│                   用户输入                        │
│     (选择Watchlist + 分析模式)                    │
└────────────────┬────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────┐
│              数据层 (Worker)                      │
│  • 获取K线数据 (AkShare)                          │
│  • 获取资金流向 (东方财富)                         │
│  • 获取新闻公告 (东财公告/DuckDuckGo/大盘背景)     │
│  • 模型: Qwen-Flash (快速)                        │
└────────────────┬────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────┐
│            信号评估 (Analyzer)                    │
│  • 资金强度评估 (相对市值)                        │
│  • 技术突破判断 (精确距离)                        │
│  • 公告重要性分级 (智能过滤)                      │
│  • 输出: 评分 (0-10分)                            │
└────────────────┬────────────────────────────────┘
                 │
        ┌────────┴────────┐
        │ 评分 >= 3?      │
        └────────┬────────┘
         ✅ 是   │   ❌ 否
        │                │
        ▼                ▼
┌──────────────┐   ┌──────────────┐
│ 决策层(Brain)│   │Worker快速分析│
│ DeepSeek-V3.2│   │ Qwen-Flash   │
│ 深度推理     │   │ 2-3句点评    │
│ 60-90秒      │   │ 5-10秒       │
└──────┬───────┘   └──────┬───────┘
       │                  │
       └──────────┬───────┘
                  ▼
          ┌────────────────┐
          │   生成报告      │
          │  • Markdown     │
          │  • 分级标注     │
          │  • 触发详情     │
          └────────────────┘
```

### 🎯 核心特性

| 特性 | 说明 |
|------|------|
| **双层架构** | Worker快速筛查 + Brain深度分析 |
| **智能触发** | 基于量化信号自动判断是否深度分析 |
| **三大Watchlist** | My/Dad/Erin 独立管理 |
| **三种模式** | Fast/Deep/Auto 灵活切换 |
| **成本优化** | 仅在关键时刻使用昂贵模型 |
| **全自动化** | Crontab定时运行，无需人工干预 |

## 命令行参数

### 完整命令格式

```bash
# Watchlist 监控模式
python main.py [--list WATCHLIST] [--analysis-mode MODE] [--no-interaction]

# 单股查询模式
python main.py --stock CODE[:NAME] [--analysis-mode MODE]
```

### 参数说明

#### `--list` (可选)

指定要分析的 Watchlist。

**可选值**：
- `my` - My Watchlist (15只股票)
- `dad` - Dad's Watchlist (10只股票)
- `erin` - Erin's Watchlist (4只股票)

**默认**：交互式选择（30秒超时，默认为`my`）

**示例**：
```bash
python main.py --list dad
```

---

#### `--stock` (可选)

启动单股查询模式，快速分析单个股票。

**格式**：`股票代码[:股票名称]` 或 `股票名称`

**智能识别功能** ⭐：
- 支持股票代码：`600519`
- 支持完整名称：`贵州茅台`
- 支持模糊匹配：`茅台` → 自动识别为贵州茅台
- 自动查找代码：输入名称自动转换为代码

**示例**：
```bash
# 方式1：仅提供代码
python main.py --stock 600519

# 方式2：提供代码和名称
python main.py --stock 600519:贵州茅台

# 方式3：仅提供名称（智能查找）⭐ 新功能
python main.py --stock 贵州茅台

# 方式4：模糊查询（智能匹配）
python main.py --stock 茅台

# 结合分析模式
python main.py --stock 中兴通讯 --analysis-mode deep
```

**智能匹配说明**：
- 如果输入模糊关键词（如"茅台"），找到多个匹配会列出选项
- 建议使用完整名称或代码以获得最佳体验
- 首次运行会自动下载股票列表并缓存（7天有效）

**特点**：
- 不需要配置 Watchlist，即查即得
- 支持所有分析模式（fast/deep/auto）
- 结果同时输出到控制台和保存到报告文件
- 报告文件命名：`HHMMSS_Stock_代码_模式.md`

---

#### `--analysis-mode` (可选)

指定分析模式。

**可选值**：
- `fast` - 快速模式（仅使用Worker）
- `deep` - 深度模式（全部使用Brain）
- `auto` - 智能模式（自动触发Brain）⭐ **推荐**

**默认**：`fast`

**示例**：
```bash
python main.py --analysis-mode auto
```

---

#### `--no-interaction` (标志)

非交互模式，适合 cron 定时任务。

**效果**：
- 不会提示选择 Watchlist
- 直接使用默认或命令行指定的参数
- 所有输出重定向到日志

**示例**：
```bash
python main.py --no-interaction --list my --analysis-mode auto
```

## 分析模式详解

### 模式对比表

| 模式 | Worker调用 | Brain调用 | 速度 | 成本 | 适用场景 |
|------|-----------|----------|------|------|---------|
| **Fast** | 100% | 0% | ⚡ 极快 (10-15秒/只) | ¥0.01/10只 | 日常监控 |
| **Auto** | 70-80% | 20-30% | ⚡ 快 (15-40秒/只) | ¥0.04-0.08/10只 | 智能决策 ⭐推荐 |
| **Deep** | 0% | 100% | 🐢 慢 (60-90秒/只) | ¥0.15/10只 | 重要决策 |

---

### Fast 模式

**特点**：
- 所有股票使用 Qwen-Flash（Worker层）
- 快速生成2-3句点评
- 成本极低

**何时使用**：
- 每日盘后快速扫描
- 只需要大致了解情况
- 预算有限

**示例输出**：
```
🤖 Worker 快速分析 [STOCK]:
**中兴通讯（000063）综合点评**
评级：中性
核心理由：资金面短期承压，技术面破位，但回购计划释放信心。
操作建议：逢低关注，等待企稳。
```

---

### Deep 模式

**特点**：
- 所有股票使用 DeepSeek-V3.2（Brain层）
- 深度分析，包含风险收益评估、具体买卖点
- 成本较高

**何时使用**：
- 周末复盘
- 准备建仓前深度研究
- 重要决策时刻

**示例输出**：
```
🧠 Brain 深度分析:
**综合评级**：看多（置信度75%）
**核心逻辑**：回购10亿元+技术刚突破年线，资金面虽弱但基本面改善
**关键信号**：
1. 大额回购释放管理层信心
2. 价格站稳年线，趋势反转
3. 估值处于历史低位

**操作策略**：
- 短线：35-36元分批建仓，止损34.5元
- 中线：目标价40元，持仓周期1-2个月
**风险提示**：资金持续流出可能引发二次探底
```

---

### Auto 模式 ⭐ 推荐

**特点**：
- 智能判断：普通股票用Worker，重要信号用Brain
- 成本可控：仅20-30%的股票触发深度分析
- 效果最优：关键时刻不错过，平时节约成本

**触发条件**（满足3分即触发）：

| 信号类型 | 评分 | 示例 |
|---------|------|------|
| **重大事件** | 3分 | 重组、并购、业绩暴增、中标 |
| **大额回购** | 3分 | 回购金额 >= 5亿 |
| **巨额资金** | 3分 | 资金流入/流出 >= 10亿 |
| **大额资金** | 2分 | 资金流入/流出 5-10亿 |
| **技术突破** | 2分 | 刚突破/跌破MA20（距离0-3%）|
| **站稳年线** | 1分 | 价格位于年线上方 |
| **回购事件** | 2分 | 回购金额 < 5亿 |
| **减持/高管变动** | 1分 | 需关注的轻度事件 |

**何时使用**：
- 日常自动化运行（Crontab）
- 希望平衡成本和质量
- 不想错过重要信号

**示例场景**：

```
案例1：思源电气
信号：资金流入8亿 + 距MA20 +1.8%
评分：2分（资金）+ 2分（技术突破）- 1分（年线）= 3分
结果：✅ 触发深度分析

案例2：中兴通讯
信号：回购10亿元
评分：3分（大额回购）
结果：✅ 触发深度分析

案例3：海尔智家
信号：常规董事会公告 + 距MA20 +1.1%
评分：0分（常规公告）+ 2分（技术突破）= 2分
结果：❌ 不触发（仅Worker快速分析）
```

### ℹ️ 问题4：看到 "Retrying request" 信息

**日志信息**：
```
14:18:43 - INFO - Retrying request to /chat/completions in 0.497778 seconds
```

**这是正常现象，不是错误！** ✅

**原因解释**：

1. **深度模型处理时间长**：
   - DeepSeek-v3.2 需要 60-90 秒才能完成深度分析
   - 远超普通 API 的响应时间（5-10秒）

2. **自动重试机制**：
   - OpenAI SDK 内置了智能重试功能
   - 当请求时间较长时，会自动重试确保成功
   - 这是为了提高稳定性，防止网络波动导致失败

3. **最终会成功**：
   - 只要最终看到分析结果，就说明一切正常
   - 重试次数一般 1-3 次

**什么时候需要担心？**

❌ **需要关注**：
```
WARNING - AI调用失败 (尝试 3/3): APITimeoutError
深度分析超时，请稍后重试
```

✅ **正常情况**：
```
INFO - Retrying request...
（最终输出完整分析报告）
```

**优化建议**：

如果经常看到重试，可以：
- 使用 `stock-check` 代替 `stock-deep`（快速模式，10秒完成）
- 网络繁忙时段避免使用深度模式
- 增加超时时间（见问题5）
