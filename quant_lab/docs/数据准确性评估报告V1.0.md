### 3. 业绩数据（营收/利润增长）⭐⭐⭐⭐⭐

**数据链路**:
```
交易所财报 → 东方财富API → AKShare → 本系统
```

**可信度分析**:
- ✅ **官方数据**: 来自上市公司季报/年报
- ✅ **时效性**: 财报披露后1小时内更新
- ⚠️ **滞后性**: 季报存在45天披露滞后（正常现象）
- ⚠️ **同比计算**: TTM营收需年化推算（Q3数据×4/3）

**已知误差来源**:
1. **季度年化推算**:
   - Q3累计营收 × 4/3 ≈ 年度营收
   - 误差范围: ±5%（季节性波动）
   - 影响: 对PS-TTM计算有轻微影响

2. **报告期选择**:
   - 当前逻辑：根据当前月份选择最近季报
   - 可能问题：财报披露延迟导致数据滞后
   - 解决方案：已缓存，定期刷新

**准确性评级**: **4.8/5.0** - 极高（扣0.2分给季度年化误差）

**适用场景**: 基本面分析、业绩趋势判断

---

### 4. 历史分位数据（PE/PB分位）⭐⭐⭐⭐

**数据链路**:
```
百度股市通（黑盒算法） → AKShare → 本系统
```

**可信度分析**:
- ✅ **数据覆盖**: 10年历史数据完整
- ⚠️ **算法黑盒**: 分位计算方法不透明
- ⚠️ **无法验证**: 无第三方数据源对比
- ⚠️ **周期性失效**: 行业周期变化时，历史分位参考价值降低

**已知问题**:
1. **算法不透明**: 不知道百度如何处理异常值、缺失值
2. **分位基准**: 不清楚是相对所有历史数据，还是仅有效交易日
3. **行业周期**: 周期股在周期顶部时，PE分位可能误导（PE低≠低估）

**误差案例**:
```
案例：煤炭股在2021年
- PE分位: 5%（历史极低）
- 实际情况: 行业周期见顶，PE低是因为市场预期利润下滑
- 误导性: 低分位≠买入机会
```

**准确性评级**: **4.0/5.0** - 高（扣1分给算法不透明）

**适用场景**:
- ✅ 辅助参考（结合基本面）
- ⚠️ 不应单独作为买卖依据
- ❌ 周期股慎用

### 5. 资金流向数据⭐⭐⭐⭐

**数据链路**:
```
Level-2逐笔数据 → 东方财富/同花顺算法 → AKShare → 本系统
```

**可信度分析**:
- ⚠️ **非官方统计**: 交易所不公布"主力资金"概念
- ⚠️ **算法差异**: 不同平台"大单"定义不同（50万? 100万?）
- ⚠️ **虚假信号**: 存在"对倒"、"假单"等市场操纵行为
- ✅ **短线参考**: A股市场广泛使用，有一定参考价值

**已知误差来源**:
1. **大单定义**:
   - 东方财富: 大单≥50万
   - 同花顺: 大单≥100万
   - 影响: 不同平台数据可能矛盾

2. **主力识别**:
   - 基于成交金额推算（非实名）
   - 无法区分机构/游资/散户
   - 可能误判

3. **对倒交易**:
   - 主力可能通过对倒制造"流入"假象
   - 系统无法识别

**准确性评级**: **3.5/5.0** - 中高（扣1.5分给非官方+算法差异）

**适用场景**:
- ✅ 短线资金情绪判断（辅助）
- ⚠️ 需结合技术面+基本面
- ❌ 不适合作为主要决策依据

## ⚠️ 已知准确性风险

### 1. 季度数据年化误差

**影响指标**: PS-TTM, 营收增长率

**误差来源**:
```python
# 当前逻辑
if report_date.endswith('0930'):  # Q3: 9个月数据
    revenue_ttm = revenue_cumulative * (4 / 3)
```

**误差范围**: ±5%（季节性波动导致）

**案例**:
```
公司: 空调制造企业
Q3累计营收: 90亿
年化推算: 90 × 4/3 = 120亿
实际年度: 130亿（Q4是销售旺季）
误差: -7.7%
```

**解决方案**:
- ⏳ 待实施：使用TTM（真实滚动12个月）而非年化推算
- ⏳ 待实施：结合历史季节性系数修正

---

### 2. 历史分位算法黑盒

**影响指标**: PE分位, PB分位

**风险**:
- 无法验证百度股市通的分位计算方法
- 可能存在数据清洗/异常值处理差异
- 周期股的分位数据容易误导

**案例**:
```
股票: 某煤炭股
PE分位: 5%（历史极低）
市场判断: 行业周期见顶，盈利下滑预期
实际情况: 低PE是"价值陷阱"，不是低估
```

**解决方案**:
- ✅ 已实施：多维度分析（不单独依赖分位）
- ⏳ 待实施：自建历史分位计算（见优化方案V1.0）
- ⏳ 待实施：添加周期股识别和特殊提示

---

### 3. 资金流向非官方统计

**影响指标**: 主力资金流入/流出

**风险**:
- "主力资金"无官方定义
- 不同平台统计口径不一致
- 可能被市场操纵（对倒）

**误差案例**:
```
同一天同一股票:
- 东方财富: 主力流入5000万
- 同花顺: 主力流出2000万
差异原因: 大单定义不同
```

**解决方案**:
- ✅ 已实施：标注数据为"辅助指标"
- ✅ 已实施：提示用户结合基本面分析
- ⏳ 待实施：多平台数据对比，识别异常值

---

### 4. 数据延迟问题

**影响场景**:
- 盘中实时决策（本系统不适用）
- 开盘/收盘瞬间（1-5分钟延迟）

**不影响场景**:
- 中长期投资决策 ✅
- 盘后复盘分析 ✅
- 周末深度研究 ✅
