## V3.0 新增功能

### 1. 多版本 Prompt 系统 ⭐

V3.0 新增 `--prompt-version` 参数，支持3种分析风格：

| 版本 | 风格 | 适用场景 |
|------|------|---------|
| `professional` | 机构研报风格 | 日常分析（默认） |
| `value_first` | 价值投资视角 | 中长期投资者 |
| `quant_hybrid` | 多因子量化评分 | 量化交易者 |

**使用示例**：
```bash
# 使用量化评分版本
python main.py --stock 002683:广东宏大 --analysis-mode deep --prompt-version quant_hybrid

# 使用价值投资版本
stock-value 广东宏大
```

---

### 2. 多周期均线系统 ⭐

新增6条均线支持：

| 均线 | 周期 | 含义 |
|------|------|------|
| MA5 | 5日 | 周线（短期动能） |
| MA10 | 10日 | 双周线（短期趋势） |
| MA20 | 20日 | 月线（中期趋势） |
| MA60 | 60日 | 季线（中期支撑） |
| MA120 | 120日 | 半年线（长期趋势） |
| MA250 | 250日 | 年线（牛熊分界） |

**均线排列检测**：
- **多头排列**：MA5 > MA10 > MA20 > MA60（强势）
- **空头排列**：MA5 < MA10 < MA20 < MA60（弱势）

### 3. 技术指标增强 ⭐

#### MACD 指标
```
MACD = DIF - DEA
DIF = EMA12 - EMA26
DEA = EMA9(DIF)
```

**信号**：
- 金叉（DIF上穿DEA）：看涨信号
- 死叉（DIF下穿DEA）：看跌信号
- 柱状线由负转正：趋势转强

#### RSI 指标
```
RSI(14) = 100 - 100/(1+RS)
RS = 平均涨幅/平均跌幅
```

**信号**：
- RSI > 70：超买（卖出信号）
- RSI < 30：超卖（买入信号）
- RSI 50-70：偏强区间
- RSI 30-50：偏弱区间

#### 量比
```
量比 = 当日成交量 / 5日平均成交量
```

**解读**：
- 量比 > 2：显著放量
- 量比 1-2：温和放量
- 量比 < 1：缩量

### 4. 缓存系统 ⭐

V3.0 引入 SQLite 缓存，显著提升性能：

| 数据类型 | 缓存时间 | 说明 |
|---------|---------|------|
| K线数据 | 4小时 | 适合盘中多次查询 |
| 资金流向 | 4小时 | 日内更新频率低 |
| 新闻公告 | 4小时 | 公告变化频率低 |
| 股票列表 | 7天 | 股票代码很少变化 |

**缓存文件位置**：`quant_lab/quant_cache.db`

**手动清除缓存**：
```bash
# 清除所有缓存
rm quant_lab/quant_cache.db

# 清除特定股票缓存（Python）
sqlite3 quant_cache.db "DELETE FROM data_cache WHERE symbol = '002683';"
```

```bash
# Watchlist 监控模式
```

### 参数说明

#### `--prompt-version` (可选) ⭐ 新增
Prompt 风格版本：
- `professional` - 机构研报风格（默认）
- `value_first` - 价值投资视角
- `quant_hybrid` - 多因子量化评分

## Prompt 版本详解

### 1. Professional（专业分析师版）

**特点**：
- 机构研报风格
- 结构化分析框架
- 客观平衡的观点

**输出格式**：
```markdown
### 综合评级
**中性偏空** | 置信度：65%

### 核心逻辑
公司营收高增长但净利润增长停滞，现金流质量堪忧...

### 关键信号
1. 价值面信号 - 估值偏高且质量不佳
2. 技术面信号 - 均线纠缠，方向不明
3. 催化剂信号 - 缺乏积极驱动

### 操作策略
- 短线：观望或谨慎试空...
- 中线：等待更佳价值买点...

### 风险提示
1. 技术面风险：...
2. 基本面风险：...
```

**适用场景**：日常分析、投资研究

---

### 2. Value First（价值投资版）

**特点**：
- 以估值和盈利质量为核心
- 长期投资视角
- 关注安全边际

**输出格式**：
```markdown
## 价值评估

### 估值分析
- PE-TTM：34.48x（偏高）
- PB：4.66x
- 估值安全边际：不足

### 盈利质量
- 营收增长：+55.9%
- 净利增长：+0.54%（严重背离）
- 现金流/净利：-0.36（负值警示）

### 投资结论
当前估值不具吸引力，建议等待更好价格...

### 中长期策略
- 理想介入区间：36.6-38.2元
- 持仓策略：波段为主
```

**适用场景**：中长期投资者、价值投资者

---

### 3. Quant Hybrid（量化评分版）

**特点**：
- 多因子量化评分
- 数值化结论
- 客观可比较

**输出格式**：
```markdown
### 多因子评分

| 因子 | 得分 | 权重 | 加权分 |
|------|------|------|--------|
| 估值 | -1 | 30% | -0.30 |
| 质量 | -1 | 25% | -0.25 |
| 动量 | 0 | 25% | 0.00 |
| 情绪 | -1 | 20% | -0.20 |
| **合计** | - | 100% | **-0.75** |

### 综合评级
**看空** | 置信度：65% | 综合得分：-0.75

### 投资结论
综合评分-0.75，落入看空区间...

### 关键观察点
1. 动量因子（中性）：短期技术面呈现积极信号...
2. 质量因子（偏弱）：核心风险在于盈利质量...

### 操作建议
- 短线策略：站稳41.5后轻仓试多，跌破39.8则观望
- 中线策略：介入区间38.15-36.6，需评估质量因子改善
- 风险控制：止损参考37.0元
```

**适用场景**：量化交易者、系统化投资

## 故障排查

### 常见问题

#### 1. "技术数据缺失"

**原因**：缓存数据过旧

**解决**：
```bash
# 清除该股票缓存
sqlite3 quant_cache.db "DELETE FROM data_cache WHERE symbol = '002683';"

# 重新查询
stock-check 002683:广东宏大
```

#### 2. API 超时

**原因**：深度分析耗时较长

**解决**：
- 正常情况：等待60-90秒
- 若频繁超时：检查网络或切换模型

#### 3. 均线数据不完整

**原因**：新股或数据不足

**解决**：
- 系统会自动使用可用数据
- MA250 需要1年历史数据
