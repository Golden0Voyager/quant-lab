# 数据源健康诊断工具 (DataSource Manager)

本工具是一个辅助性的监控脚本，用于诊断和评估系统中各个第三方金融 API 的健康状况。

## 📂 文件位置
- 脚本路径: `tests/diagnose_data_sources.py`

## 🎯 核心功能
1. **API 质量监控**: 统计雪球、东财、新浪等多个数据源的请求成功率。
2. **多源降级测试**: 模拟实际运行中的数据抓取流程，验证当首选数据源失效时，系统是否能自动切换到备用源。
3. **数据完整度检查**: 验证抓取到的 JSON 数据中 PE、PB、市值等关键字段是否完整。
4. **生成体检报告**: 在终端输出直观的成功率对比表，帮助开发者判断当前网络环境或 IP 状态。

## 🚀 使用场景
- **排查 N/A 问题**: 当大量股票的估值显示为 "N/A" 时，运行此脚本可以快速确定是哪个 API 挂了。
- **开发新维度**: 在引入新的抓取逻辑前，用此工具评估数据源的稳定性。
- **日常维护**: 建议在网络状况不佳时运行，以决定是否需要更换 API 策略。

## 🛠 运行方法
在项目根目录下执行：
```bash
uv run python tests/diagnose_data_sources.py
```

## 📊 输出示例
```

✅ eastmoney_spot  : 100.0% (3/3)
```

## 📝 注意事项
- 本工具为**诊断工具**，主程序分析逻辑并不强制依赖它。
- 频繁运行此工具可能会导致 IP 被第三方接口短暂封禁，建议仅在排障时使用。
